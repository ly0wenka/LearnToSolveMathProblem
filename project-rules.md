# Project Rules

## Topic

Bachelor thesis topic:

`Проєктування та розробка програмного забезпечення для навчання з розв'язання математичних задач`

## Object and subject

- Object: the process of digital support for learning mathematics.
- Subject: methods, models, and software components for interactive step-by-step solving of mathematical tasks.

## Goal

Design and implement a software system that helps learners solve mathematical problems through guided steps, self-checking, and progress analytics.

## Numbered tasks

1. Analyze the domain and learning needs of pupils, students, and tutors.
2. Define functional and non-functional requirements for the learning system.
3. Design the architecture, data model, and API interaction of the application.
4. Implement the frontend and backend components of the system.
5. Prepare the database structure for scalable storage of users, tasks, and results.
6. Provide a basis for testing, progress tracking, and future recommendation logic.

## Expected conclusions mapping

1. The domain and target audience needs are formalized.
2. The system requirements are identified.
3. The software architecture and database model are designed.
4. The educational web application prototype is implemented.
5. The backend and storage structure for scaling are prepared.
6. The project foundation for analytics and adaptive learning is created.
